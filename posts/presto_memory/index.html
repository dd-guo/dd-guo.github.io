No Where
Themed by Diary.
0001-01-01 00:00

code

  • builder.put(RESERVED_POOL, new MemoryPool(RESERVED_POOL, config.getMaxQueryMemoryPerNode()));
  • builder.put(SYSTEM_POOL, new MemoryPool(SYSTEM_POOL, systemMemoryConfig.getReservedSystemMemory()));
  • general_pool
long maxHeap = Runtime.getRuntime().maxMemory();
maxMemory = new DataSize(maxHeap - systemMemoryConfig.getReservedSystemMemory().toBytes(), BYTE);
DataSize generalPoolSize = new DataSize(Math.max(0, maxMemory.toBytes() - config.getMaxQueryMemoryPerNode().toBytes()), BYTE);
builder.put(GENERAL_POOL, new MemoryPool(GENERAL_POOL, generalPoolSize));

explain

  • RESERVED_POOL = config.properties query.max-memory-per-node
  • SYSTEM_POOL = config.properties resources.reserved-system-memory (default Runtime.getRuntime().maxMemory() * 0.4 即 0.4 * Xmx)
  • GENERAL_POOL = Xmx - RESERVED_POOL - SYSTEM_POOL

description

  • GENERAL_POOL the memory pool used by the physical operators in a query
  • SYSTEM_POOL mostly used by the exchange buffers and readers/writers
  • RESERVED_POOL for running a large query when the general pool becomes full (only used by one query), 条件
    • GENERAL_POOL有节点出现阻塞节点（block node）情况，即该node内存不足
    • RESERVED_POOL没有被使用

action

  1. 整体presto给30G内存，requests是30G，limits是32G
  2. 可以把reserved_pool分配到5G; system_pool 10G; general_pool 15G

参数设置

yaml

  • limits: 35G
  • requests: 32G

config.properties

  • query.max-memory=80GB
  • query.max-total-memory=120GB
  • query.max-memory-per-node=10GB
  • query.max-total-memory-per-node=20GB
  • memory.heap-headroom-per-node=6GB
  • resources.reserved-system-memory=10GB

jvm.config

  • -Xmx40G
  • -Xms32G
  • -XX:ConcGCThreads=6
  • -XX:ParallelGCThreads=8
  • -XX:G1ReservePercent=10
  • -XX:InitiatingHeapOccupancyPercent=50
